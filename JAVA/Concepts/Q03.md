## 🧪 The Question

**Code:**

```java
abstract class Shape {
    abstract double area();

    void display() {
        System.out.println("Area is: " + area());
    }
}

class Circle extends Shape {
    double radius;

    Circle(double radius) {
        this.radius = radius;
    }

    @Override
    double area() {
        return 3.14 * radius * radius;
    }
}
```

### Questions Asked
1. Can you do `Shape s = new Shape()`? Why or why not?
2. What is the difference between an `abstract` method and a regular method?
3. What is the purpose of `this.radius = radius` in the constructor?

---

## 📝 Student Answers

1. No, we cannot do `new Shape()` because `Shape` is abstract — but we **can** do upcasting: `Shape s = new Circle(45)`
2. An `abstract` method has only a declaration, like a blueprint, so subclasses can define it according to their own need by overriding it
3. `this` is used to point to the current object and to **differentiate between the parameter and the instance field** when both have the same name

---

## 📊 Answer Analysis

### Q1 — Can you do `Shape s = new Shape()`?

**Status: ✅ Correct**

```java
Shape s = new Shape();    // ❌ Compile Error — abstract class cannot be instantiated
Shape s = new Circle(45); // ✅ Valid — upcasting with a concrete subclass
```

| Reference | Object | Valid? |
|-----------|--------|--------|
| `Shape s` | `new Shape()` | ❌ Abstract — cannot instantiate |
| `Shape s` | `new Circle(45)` | ✅ Upcasting — perfectly fine |

> 🔑 An abstract class is a **blueprint only**. You can use it as a reference type but can never create an object directly from it.

---

### Q2 — Abstract Method vs Regular Method

**Status: ✅ Correct**

| Type | Has Body? | Must Override in Subclass? |
|------|-----------|---------------------------|
| `abstract` method | ❌ Declaration only | ✅ Yes, mandatory |
| Regular method | ✅ Full body | ❌ Optional |

```java
abstract double area();          // ← No body, just a declaration (blueprint)
void display() { ... }           // ← Has a full body, ready to use
```

> 🔑 If a class has even **one** abstract method, the **entire class must be declared `abstract`**.

---

### Q3 — Purpose of `this.radius = radius`

**Status: ✅ Correct**

Both the constructor **parameter** and the **instance field** share the same name `radius`. The `this` keyword removes the ambiguity:

```java
Circle(double radius) {
    this.radius = radius;
//  ↑ instance field    ↑ constructor parameter
}
```

| | Name | Refers To |
|--|------|-----------|
| `this.radius` | `radius` | Instance variable of the object |
| `radius` | `radius` | Parameter passed to constructor |

```java
// ❌ Without this — assigns parameter to itself, field never set!
radius = radius;

// ✅ With this — parameter value correctly saved into the object's field
this.radius = radius;
```

## 💡 Key Concepts to Remember

| Concept | Description |
|---------|-------------|
| **Abstract Class** | Cannot be instantiated directly; serves as a blueprint |
| **Abstract Method** | Has no body; must be implemented by all concrete subclasses |
| **Upcasting** | `Shape s = new Circle(45)` — valid way to use an abstract reference |
| **`this` keyword** | Refers to the current object; resolves conflict between field and parameter of same name |
| **Concrete Class** | A non-abstract class that implements all inherited abstract methods |

---

## ⚠️ Common Mistakes to Avoid

```java
// ❌ Cannot instantiate abstract class
Shape s = new Shape();

// ✅ Use a concrete subclass with upcasting
Shape s = new Circle(45);

// ❌ Without this — parameter assigned to itself
radius = radius; // field never updated!

// ✅ With this — field correctly set
this.radius = radius;
```
