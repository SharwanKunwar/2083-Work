## 🧪 The Question

**Code:**

```java
interface Flyable {
    void fly();
}

interface Swimmable {
    void swim();
}

abstract class Animal {
    abstract void sound();
    void breathe() {
        System.out.println("Breathing...");
    }
}

class Duck extends Animal implements Flyable, Swimmable {
    public void fly()   { System.out.println("Duck flying!"); }
    public void swim()  { System.out.println("Duck swimming!"); }
    public void sound() { System.out.println("Quack!"); }
}
```

### Questions Asked
1. Can `Animal` implement multiple abstract classes? Why or why not?
2. What is the **key difference** between an `interface` and an `abstract class`?
3. Why does `Duck` use both `extends` and `implements` — and can the order be swapped?

---

## 📝 Student Final Answers

1. `Animal` **cannot implement** abstract classes at all — the word `implement` only applies to **interfaces**. You `extend` classes, and Java only allows extending **one** class (abstract or not).

2. An abstract class can have instance, static, and final fields + regular methods + constructor. An interface can only have `public static final` fields and abstract methods by default (Java 8+ allows `default` methods). Most importantly, a class can implement **multiple interfaces** but extend only **one** abstract class.

3. `Duck` uses `extends Animal` because Duck **IS AN** Animal, and `implements Flyable, Swimmable` because Duck **CAN** fly and swim. The order **cannot be swapped** — `extends` must always come before `implements`, otherwise it's a compile error.

## 📊 Answer Analysis

### Q1 — Can `Animal` implement multiple abstract classes?

**Status: ✅ Correct**

The precise answer lies in the keyword itself:

| Action | Target | Allowed? |
|--------|--------|----------|
| `implements` | Interface | ✅ Multiple allowed |
| `extends` | Abstract Class | ✅ Only **one** |
| `implements` | Abstract Class | ❌ Wrong keyword — never valid |

```java
// ❌ Cannot implement a class — wrong keyword
abstract class Animal implements AbstractA { } // Compile Error!

// ✅ Can extend ONE abstract class
abstract class Animal extends AbstractA { }

// ❌ Cannot extend multiple classes
abstract class Animal extends AbstractA, AbstractB { } // Compile Error!

// ✅ Abstract class CAN implement multiple interfaces
abstract class Animal implements Flyable, Swimmable { }
```

> 🔑 **You `implement` interfaces. You `extend` classes. Java allows only one class to be extended.**

---

### Q2 — Interface vs Abstract Class

**Status: ✅ Correct**

| Feature | Abstract Class | Interface |
|---------|---------------|-----------|
| Fields | Any type (instance, static, final) | `public static final` only |
| Methods | Abstract + regular both | Abstract by default (Java 8+ allows `default`) |
| Constructor | ✅ Can have | ❌ Cannot have |
| Inheritance | Only **one** | **Multiple** allowed |
| Purpose | *"IS-A"* relationship | *"CAN-DO"* capability |

```java
// Abstract class — IS-A relationship
abstract class Animal {
    String name;                    // instance field ✅
    abstract void sound();          // abstract method
    void breathe() { ... }          // regular method ✅
}

// Interface — CAN-DO capability
interface Flyable {
    public static final int MAX_HEIGHT = 1000; // only this type of field
    void fly();                                // abstract by default
}
```

> 🔑 **Biggest difference** — A class can implement **multiple interfaces** but extend only **one** abstract class.

---

### Q3 — Why `extends` + `implements` & Can Order Be Swapped?

**Status: ✅ Correct**

```java
class Duck extends Animal implements Flyable, Swimmable { }
//          ↑ IS-AN Animal       ↑ CAN fly and swim
```

| Keyword | Meaning | Relationship |
|---------|---------|--------------|
| `extends Animal` | Duck is an Animal | **IS-A** |
| `implements Flyable` | Duck can fly | **CAN-DO** |
| `implements Swimmable` | Duck can swim | **CAN-DO** |

```java
// ✅ Correct — extends MUST come before implements
class Duck extends Animal implements Flyable, Swimmable { }

// ❌ Wrong order — Compile Error!
class Duck implements Flyable, Swimmable extends Animal { }
```

> 🔑 **Order CANNOT be swapped.** `extends` always comes before `implements` — Java rule!

## 💡 Key Concepts to Remember

| Concept | Rule |
|---------|------|
| `implements` | Only for **interfaces** |
| `extends` | Only for **classes** (one only) |
| Multiple interfaces | ✅ A class can implement many |
| Multiple classes | ❌ Java does not allow |
| `extends` before `implements` | ✅ Always — Java syntax rule |
| Abstract class + interfaces | ✅ Abstract class can implement multiple interfaces |

---

## ⚠️ Common Mistakes to Avoid

```java
// ❌ Cannot implement a class
class Duck implements Animal { }

// ❌ Cannot extend multiple classes
class Duck extends Animal, Creature { }

// ❌ Wrong order
class Duck implements Flyable extends Animal { }

// ✅ Correct
class Duck extends Animal implements Flyable, Swimmable { }
```
