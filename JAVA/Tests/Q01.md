## 🧪 Question 1 — Inheritance + Method Overriding

### Code

```java
class Animal {
    String name = "Animal";

    void sound() {
        System.out.println("Some sound");
    }
}

class Dog extends Animal {
    String name = "Dog";

    @Override
    void sound() {
        System.out.println("Bark");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal a = new Dog();
        a.sound();
        System.out.println(a.name);
    }
}
```

### Questions Asked
1. What is the output of this program?
2. *Why* does `a.name` print what it prints? (Explain the concept behind it)
3. What is the term for `Animal a = new Dog()`? And why is it useful?

### Q2 — Concept Behind `a.name`

**Status: ❌ Incorrect**

> 🔴 **Fields are NOT polymorphic in Java. Only methods are.**

- **Methods** → resolved at **runtime** based on the actual object (`Dog`) → prints `Bark`
- **Fields** → resolved at **compile time** based on the reference type (`Animal`) → prints `Animal`

This is called **Field Hiding**. `Dog.name` hides `Animal.name`, but since the reference is of type `Animal`, Java uses `Animal`'s field.

```java
Animal a = new Dog();
a.sound(); // Runtime decision  → Dog's method → "Bark"
a.name;    // Compile-time decision → Animal's field → "Animal"
```

## 💡 Key Concepts to Review

- **Method Overriding** — Subclass provides a specific implementation of a method already defined in the parent class.
- **Field Hiding** — Fields are not overridden; they are hidden. The reference type determines which field is accessed.
- **Upcasting** — Assigning a subclass object to a superclass reference (`Animal a = new Dog()`).
- **Runtime Polymorphism** — The JVM decides at runtime which method to call based on the actual object type.
- **Compile-time Resolution** — Fields and static methods are resolved at compile time based on the reference type.
