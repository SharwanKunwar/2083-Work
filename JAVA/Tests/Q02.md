## 🧪 The Question

**Look at this code:**

```java
class BankAccount {
    public double balance = 5000;

    public void deposit(double amount) {
        balance += amount;
    }
}

public class Main {
    public static void main(String[] args) {
        BankAccount acc = new BankAccount();
        acc.balance = -99999;  // ← This line
        System.out.println(acc.balance);
    }
}
```

### Questions Asked
1. What is the **problem** with this code?
2. How would you **fix it** using Encapsulation? *(write the corrected class)*
3. What are the **two things** Encapsulation always uses together to protect data?

## 📊 Answer Analysis

### Q1 — Problem with the Code

**Status: ❌ Missed It**

The problem is that `balance` is `public`, so **anyone can directly set it to any invalid value** like `-99999` with zero restriction.

```java
acc.balance = -99999; // ← Nothing stops this! Big problem for a bank 🏦
```

> 🔑 A bank account balance should **never** be set to an arbitrary negative value from outside. The class itself should **control** how its data is modified.

---

### Q2 — Fix Using Encapsulation

**Status: 🟡 Partial Credit**

Making the field `private` is the **first step ✅**, but the fix is incomplete without validation and controlled access.

**Corrected Class:**

```java
class BankAccount {
    private double balance = 5000; // ← Step 1: Make it private

    public void deposit(double amount) {
        if (amount > 0)            // ← Step 2: Add validation
            balance += amount;
    }

    // Getter — controlled read access
    public double getBalance() {
        return balance;
    }

    // No direct setter — balance should only change via deposit()
}
```

> ✅ Now `acc.balance = -99999` causes a **compile-time error**. The class is fully in control of its own data!

## 💡 Key Concepts to Remember

| Concept | Description |
|---------|-------------|
| **Encapsulation** | Wrapping data and methods together, hiding internal state |
| **`private` field** | Cannot be accessed directly from outside the class |
| **Getter** | A `public` method that returns the value of a private field |
| **Setter** | A `public` method that sets the value with optional validation |
| **Data Validation** | Checking input before allowing it to change internal state |

---

## ⚠️ Common Mistake to Avoid

```java
// ❌ Bad — no encapsulation
public double balance = 5000;

// ✅ Good — encapsulated with validation
private double balance = 5000;
public double getBalance() { return balance; }
public void deposit(double amount) {
    if (amount > 0) balance += amount;
}
```
