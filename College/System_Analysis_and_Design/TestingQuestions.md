# 🧪 System Analysis & Design - Important 10 Marks Questions (Testing Unit)

---

## 📌 1. Software Testing & Levels of Testing

### Question: What is software testing? Explain its importance in system development. Also, describe different levels of testing.

---

👉 **Answer should include:**

### 🔹 Software Testing
- Definition of software testing  
- Purpose: finding errors/bugs  
- Ensuring system quality and reliability  

### 🔹 Importance of Software Testing
- Improves system quality  
- Detects errors before delivery  
- Reduces maintenance cost  
- Increases user satisfaction  

### 🔹 Levels of Testing
- Unit Testing  
- Integration Testing  
- System Testing  
- Acceptance Testing  

---

## 📌 2. Black Box vs White Box Testing

### Question: Differentiate between black box testing and white box testing with suitable examples. Explain their advantages and disadvantages.

---

👉 **Answer should include:**

### 🔹 Black Box Testing
- Meaning (no internal code knowledge)  
- Focus on input/output  
- Done by testers  
- Example: login system testing  

### 🔹 White Box Testing
- Meaning (internal code knowledge required)  
- Focus on logic, loops, conditions  
- Done by developers  
- Example: testing loop conditions  

### 🔹 Differences
- Internal vs external view  
- User vs developer perspective  
- Functionality vs code structure  

### 🔹 Advantages & Disadvantages
- Black box: easy, user-focused / limited internal coverage  
- White box: detailed logic check / requires coding knowledge  

---

## 📌 3. Verification vs Validation in System Development

### Question: Differentiate between verification and validation in system development. Explain their importance in system quality assurance.

---

👉 **Answer should include:**

### 🔹 Verification
- “Are we building the system right?”  
- Checks documents, design, and code  
- Done during development  

### 🔹 Validation
- “Are we building the right system?”  
- Checks actual working system  
- Done after development  

### 🔹 Differences
- Static vs dynamic process  
- Development phase vs final product testing  
- Document check vs system execution  

### 🔹 Importance
- Ensures correctness of system  
- Reduces errors and failure risk  
- Improves user satisfaction
