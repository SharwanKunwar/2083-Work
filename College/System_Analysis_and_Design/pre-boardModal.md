- Collects and understands user requirements.

- Prepares system documents and reports

## 4. Explain different types of feasibility assessment to be taken.

# 📊 Feasibility Assessment

Feasibility assessment is the process of evaluating whether a proposed system is practical and beneficial.

## 🔍 Types of Feasibility

### 1. Technical Feasibility
- Checks availability of technology, hardware, and skills  

### 2. Economic Feasibility
- Compares cost with expected benefits  

### 3. Operational Feasibility
- Determines if the system will work in the organization  

### 4. Legal Feasibility
- Ensures compliance with laws and regulations  

### 5. Schedule Feasibility
- Checks if the project can be completed on time  

## ✅ Conclusion
It helps in deciding whether to proceed with the system or not.

## 6. Define user interface. Explain different types of interfaces that can be implemented in modern system . What are different methods to manage maintenance const.

## 📌 1. User Interface (UI)

A **User Interface (UI)** is the point of interaction between a user and a computer system. It allows users to input data and receive output in a simple and understandable way.



## 🧭 2. Types of Interfaces in Modern Systems

### 1. Command Line Interface (CLI)
- Users type commands to interact with the system  
- Example: Linux terminal  
- Fast but requires technical knowledge  

### 2. Graphical User Interface (GUI)
- Uses icons, buttons, menus, and windows  
- Easy to use and user-friendly  
- Example: Windows, mobile apps  

### 3. Menu-Driven Interface
- Users select options from menus  
- Example: ATM machines  

### 4. Touch Interface
- Uses touch gestures like tap, swipe, and pinch  
- Example: Smartphones and tablets  

### 5. Voice-Based Interface
- Uses voice commands for interaction  
- Example: Google Assistant, Siri  

---

## 🔧 3. Methods to Manage Maintenance Cost

### 1. Preventive Maintenance
- Regular system checks and updates  
- Prevents future failures  

### 2. Corrective Maintenance
- Fixing errors and bugs after they occur  

### 3. Adaptive Maintenance
- Modifying system for new environments or requirements  
- Example: OS or hardware upgrades  

### 4. Perfective Maintenance
- Improving system performance and features  

---

## ✅ Conclusion
User Interface helps users interact with systems easily, and different interface types improve usability. Maintenance methods help reduce cost and ensure system efficiency and reliability.

## 7. What are the phases that is incorporated within implementation. Differentiate between black box and white box testing.


Implementation is the process of converting a designed system into a working system.

### 🔹 1. Coding
- Writing program code using programming languages  
- Converts design into software  

### 🔹 2. Testing
- Checking the system for errors and bugs  
- Ensures system works correctly  

### 🔹 3. Installation
- Installing software on user systems or servers  
- Making the system ready for use  

### 🔹 4. Conversion
- Switching from old system to new system  
- Methods: direct, parallel, phased, pilot  

### 🔹 5. Documentation
- Preparing user manuals and technical documents  
- Helps users and developers understand the system  

### 🔹 6. Maintenance
- Fixing errors and improving system after deployment  

---

## 🧪 2. Difference Between Black Box and White Box Testing

| Feature | Black Box Testing | White Box Testing |
|----------|------------------|------------------|
| Definition | Testing without knowing internal code | Testing with knowledge of internal code |
| Focus | Input and output | Internal logic and code structure |
| Done by | Testers / end users | Developers |
| Programming knowledge | Not required | Required |
| Purpose | Check functionality | Check code quality and logic |
| Example | Login system testing | Checking loops and conditions in code |

---

## ✅ Conclusion
Implementation includes coding, testing, installation, conversion, documentation, and maintenance. Black box testing focuses on output behavior, while white box testing focuses on internal code structure.
